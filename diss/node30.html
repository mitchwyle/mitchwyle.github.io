regular expressions

 

The implementation of regular expressions in the Pasadena system is encapsulated in the module match (see figure 4.2). The routines in match provide an implementation of an extended class of regular expressions [Enge88]. The pattern matching functions are used frequently by several modules in the main daemon program and the module is imported by other external programs as well. Regular expressions are compiled into finite-state machines for matching text patterns efficiently. The source (non-compiled) regular expressions, i.e. the inputs that a subscriber would compose in a filtering stage, are defined as pure ASCII character strings. Larger expressions are built up from single-character regular expressions according to the following rules.

A concatenation of REs matches a concatenation of text strings, but does not cross paragraph boundaries.

Two REs on consecutive lines match REs in separate paragraphs. Two REs separated by a blank line match messages containing either RE. This formulation is therefore a disjunctive normal form.
