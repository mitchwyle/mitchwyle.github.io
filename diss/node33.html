Retrieval Methods Tested

In order to test components of the system during its initial implementation, simple Boolean indexing and retrieval methods were developed together with the main daemon program. The retrieval strategy in these Boolean algorithms used simple coordination level matching [Salt83] to generate RSVs. A simple variation of Boolean coordination level matching using normalization [Gige89] was contrasted to standard coordination level matching. The results of these very early experiments, which were primarily used for exercising the system, verified well-known results from the literature, namely that normalization of Boolean queries provides slightly better retrieval performance.

In addition to these simple Boolean algorithms, several different IR algorithms were implemented and tested in the Pasadena system. These methods implement algorithms based on the equations presented in the second chapter. To find out which components of the methods tested were responsible for performance differences, we systematically ran experiments whereby one component of one method was varied, then measured the retrieval performance of the variation relative to the baseline algorithms and to each other. These variations include the extent of word reduction, the type of features used in indexing, the text comparison RSV function, and the various term weighting methods from sections 2.2 and 2.3.

The baseline method against which most algorithms were compared uses Porter [Port80] reduced words as indexing features, equation 2.9 for feature weighting (), and the standard cosine function for RSV calculation. Henceforth we shall refer to this method as psw (a mnemonic for `` Porter- stemmed words).''

A simple variation of the psw method uses S-stemmed words for features. In this method, only certain plural words are reduced to their singular forms; this method is implemented simply by using the first two rules of Porter's reduction algorithm. Feature weighting and RSV calculation are identical to those of psw. This method is referred to as ssw, a mnemonic for `` S- stemmed words.''

Another variation of the psw method uses an indirect similarity measure (see section 2.2, equation 2.11) instead of the cosine RSV function. This method is called pwi, a mnemonic for `` Porter reduced words, indirect similarity.''

The first published results from the experiments run in the Pasadena system compared the use of N-grams as indexing features to S-stemmed words [Wyle91]. The method ng (a mnemonic for `` N- grams''), uses the indirect similarity RSV function in equation 2.11 and overlapping N-grams for indexing features. The standard (equation 2.9) feature weighting function is used for term weighting.

A variation of the N-gram method using the cosine direct similarity instead of the indirect similarity method was also implemented. This variation is called ngc for `` N- gram cosine.''

For a detailed description of these N-gram methods, including the motivations for and reasoning behind the use of N-grams and indirect similarity, see [Teuf89] and [Wyle91]. In the latter work, the performance differences resulting from these variations of ng and psw were compared using the Pasadena system. Some of the results in that work were inconclusive because of high error probabilities resulting from insufficient data. New performance data contrasting these algorithms is reported here (see below, section 4.5 and appendix B).

Another method implemented and tested in the Pasadena system is based on the use of simple phrases [Faga87] as indexing features. In this method, combinations of Porter reduced words from the query are extended to phrases, and these phrases along with all single words are used as indexing features. is used to weight the features, and the cosine function is used for the RSV calculation. This method is called sph, a mnemonic ``for simple phrases.''

The phrase generation method described in section 3.4 was, of course, also implemented in the Pasadena system. The Porter reduction algorithm is used on all words. The resulting set of individual words combined with phrases constructed from the symmetric extension of high-noise words are used as indexing features for the similarity measures. The indirect similarity RSV function is used. Three different feature weighting functions were implemented, the standard (equation 2.9), a simple variation of the standard function where the message frequency is hyperbolically normalized (equation 2.10), and feature weights based on their inverse message density (equation 2.16). The first variation of this method using is called nph, for `` noise discriminated words to phrases.'' The variation of this method that uses normalized message frequency feature weighting is called npn, a mnemonic for `` noise discriminated words to phrases, normalized.'' The final variation using both message frequency normalization and the inverse message density factor is called npi for `` noise discriminated words to phrases with inverse message density.''

A variation of the proposed flexible text matching system using different context granularities proposed by Salton and Buckley [Salt91] and described in section 2.3 was also implemented in the Pasadena system. The term weighting function 2.12 and retrieval function 2.13 were implemented using two granularity levels, the so-called global context (entire message) and the paragraph context. The mnemonic for this method is sps for `` Salton paragraph system.''

Finally, a method incorporating the inverse feature density term weighting (equation 2.14) that uses statistics based on paragraph types (section 2.3) was implemented using the standard cosine RSV function and Porter stemmed words as indexing features. This method is labelled ipd, a mnemonic for `` inverse feature in paragraph density.''

All of these algorithms are constructed from variations of the same basic modules and are structured identically (see figures 4.5 and 4.6). They all use similar data structures for indexing and ranking, but the retrieval algorithms differ significantly.

In this section we discussed the implementation of the Pasadena system, highlighting some of its unique features and interesting aspects of the system realization. Pasadena consists of a relatively small core daemon program that coordinates and manages the scheduling, execution, and data management of several other semi-independent components. This daemon process also handles subscribers profile management including executing user commands. A separate selective information extraction system querys remote databases aperiodically based on the information flow parameters discussed in section 2.1. The main daemon program has a well-defined interface to different IR indexing and retrieval algorithms that are implemented as a set of separate executable programs. The software architecture and implementation of twelve major algorithms tested in the system was presented briefly. We come now to the results of the numerous experiments run in the system to test the retrieval performance of these twelve algorithms.
