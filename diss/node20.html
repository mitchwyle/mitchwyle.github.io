Sifting in Stages

       

We now introduce the notion of using separate retrieval functions instead of a single information filtering process. We call these functions filtering stages and define some parameters for their application and combination. Factoring out levels of information filtering from the overall IR process into independent layers is advantageous not only because it enhances the flexibility  and retrieval capabilities for subscribers, but also for managing network  bandwidth  and computational resources. Decomposing  the filtering  process  into stages  gives a subscriber much more flexibility in formulating   a profile. Specific aspects of the information need are handled separately, which provides a structure in the profile. Evidence from the literature [Crof91] indicates that such a structure is advantageous to the retrieval process. Stages also give a system architect more degrees of freedom in constructing a filtering service. Separate stages could, for example, be run on separate processors or machines. Network bandwidth can be optimized by positioning filtering stages providing a large data  reduction    topologically (in a networking sense) close to the information source. Storage and other computational resources are used more efficiently because only those information sources with possibly relevant information need be tracked by a server.

Well-defined search  criteria , operating on formal , specific  fields  of WAN messages, are used to extract a promising  set  of messages  for the IR ranking  process . Specific patterns are used to retrieve or eliminate messages independently of their ranking. Paragraph types  (see section 2.3) are then designated in order to deliver only relevant full-text sections or the appropriate message surrogates to subscribers.

We turn now to the development of a model for an information  server . We model an information server (IS)  as a dynamic  intermediate  repository  of   WAN messages whose purpose is to deliver all and only those messages in the Matrix which a subscriber prefers to all other available messages. The    system's    subscribers are the information sinks and the heterogeneous WAN information providers are the information sources. In our model, an IS includes methods for optimally and efficiently gathering information from WAN sources and delivering the most ``relevant,'' that is the messages preferred to all other messages, to its subscribers.

An IS is composed of a set of system  parameters , state  information  for tracking remote    sources    in the Matrix , a set of subscriber profiles , a momentary    database of locally cached   WAN messages, and some auxiliary information  such as article indexes and feature  statistics . Each user profile contains some general  parameters about the subscriber such as how to reach him or her, and specific elements for modelling an information  need :

  • First and foremost, a profile contains a set of zero or more topics about which the subscriber wishes to be informed. We call these topics queries;   it is important to distinguish these queries from the usual query texts associated with other types of information systems. Queries in this model contain several parameters in addition to a text. These query parameters are described in detail below.
  • The second element in a profile is a sequence of extended regular expressions [Enge88] (see section 3.3) in disjunctive normal form used to model a subscriber's future  interest  in possible  sources of information. These regular expressions are used to match the titles or descriptions of new sources of information as they become available. They are not meant to match the actual information within the new sources; rather the name or description of new WAN databases which are continually becoming available in the Matrix.
  • In order to make the formulation, storage, and processing of queries easier, a profile also contains global defaults for all non-text parameters in each query.

A query is composed of some user and system identification         information, a query  text  of unlimited length which the subscriber may change frequently, and the following parameters:

  • The number of messages in a retrieval set to be delivered;      
  • The time interval between deliveries;    
  • A set of one or more remote WAN database categories:       These categories map directly to existing, distinct WAN  databases to be used as a retrieval  source;
  • Zero or more inclusion  patterns ; these             patterns are also extended regular expressions in disjunctive normal form. These inclusion patterns are designed to be a highly specific template which will match a very small set of messages that the subscriber prefers above all other possible messages, regardless of their similarity to the text portion of the query. These patterns are typically the name of a specific individual or a very exact (and therefore long) phrase;
  • Zero or more exclusion patterns; these patterns are once again             extended regular expressions OR'd together in disjunctive normal form. Exclusion patterns are designed to filter out specific messages which have a high syntactic similarity to a query text but little semantic correspondence. For example, if a subscriber is interested in pixel orientation in object recognition systems, she would not want to see messages about object oriented programming languages or the recognition received by object oriented database companies;
  • A set of one or more paragraph types [Hort87] to be   be   displayed in the messages sent from the IS.

An IS uses the union of all database  categories of all queries of all subscribers as the sources in the Matrix  for messages to cache in its  local database DB. As the database categories change in the queries of the user profiles in an IS, the sources queried by the IS change accordingly. DB also changes as older messages are deleted from the IS cache and as new messages arrive.

We shall now consider some properties of the sets resulting from the application of filtering stages and the relationships among these sets. These properties can be used to check the viability and operating bounds of filtering functions after their application. As we shall see in the following discussion, the result of these tests can then be used to signal warnings and suggest changes to filtering functions based on undesired results.

We examine three classes  of overlap  among the sets of messages resulting from separate filtering stages. A pair of message sets can be:  

  • disjoint: Two sets of messages are disjoint when they contain no   elements in common .
  • semi-orthogonal:   Two   sets are semi-orthogonal when they are either disjoint, or contain some elements in common, but are not identical, .
  • unrestricted: There   is no restriction on the sets; they may be identical, overlapping, or disjoint.
Obviously, these classes are hierarchical. All pairs of disjoint sets are semi-orthogonal and all pairs of semi-orthogonal sets are unrestricted.

The filtering stages constructed by the inclusion pattern and the complement of the exclusion pattern (described above) are designed to produce either identical or semi-orthogonal answer sets. If the answer sets are disjoint, their sequential application would result in a completely empty answer set. Although this situation may indeed be desired by the subscriber, it can often be an indication that the patterns composing one or both of the filtering stages is too broad or too specific. In such a situation, a warning can be issued by the system to indicate that one or both of the filtering stages may need changes. Similarly, the filtering stage resulting from the query text and an IR algorithm is designed to be semi-orthogonal to the set resulting from the application of the exclusion  stage. If the resulting sets of the application of these stages are disjoint, it is likely that the exclusion stage is too broad.

A final post-processing filtering action  is defined by a query-specific parameter that specifies a list of paragraph  types to be included in each message delivered to s. It also indicates the order   in which these paragraphs should appear when the message is delivered. The subscriber can thereby filter out unwanted header  paragraphs  and directly control the size and order of the summarizing paragraphs of each message delivered.

It is preferable to rank and retrieve only information items that have potential  relevance  to a query because of the large and exponentially growing number of messages available. The reason for this preference is simply an efficiency issue. Narrowing down the set of messages to be ranked using the stages through is therefore conceptually superior to a simple single-stage ranking method. The explicit elimination of messages containing a specific pattern directly reduces the fallout and therefore enhances precision performance.

There is some evidence to support the hypothesis that filtering in stages is more effective than one-stage paradigms [Salt90], [Mans92]. Salton and Buckley, though not concerned with filtering per se propose a two-stage similarity measure in which the similarity between a query text and message is calculated by applying two separate functions in sequence. In discussing the benefits of this method, they speculate that ``this precision performance may be improvable by using additional matching criteria for low level constructs.'' In other words, by adding another stage to their overall similarity measure. In the same work, they also conjecture that ``restricting the collection search to items located within the same text cluster might also produce somewhat enhanced performance measurements.'' It seems that they find filtering in stages an idea worthy of further pursuit. Mansfield et. al. use hardware methods for sequential filtering stages. They claim that ``Efficient processing of complex associative search queries enables services that are impractical using conventional database system approaches.'' Their use of filtering stages is implemented in hardware and is applied to optimize distributed database applications in telephone and cable television networks. Filtering in this manner is in any case more flexible for network and computing resource allocation [Lee88].

With the growth and increased use of the Matrix, there has been a resurgence in studies of methods for so-called resource  discovery , a concept which has migrated from the field of library  science to the WAN literature in computer science [Danz91]. Use of the query-specific profile parameter which matches new database categories as new WAN databases become available addresses precisely this issue: when an IS gains access to new source databases in the Matrix, subscribers whose queries contain a pattern which matches the text of the name or description of these new databases are automatically informed of the existence of the new source. Thus an IS can facilitate resource discovery among its subscribers in a similar fashion to the way messages themselves are disseminated. The pattern for matching the descriptions of new databases as they become available is another parameter designed to satisfy conjectural preferences normally associated with the classical recall measure. An IS does not necessarily add new databases to queries automatically. Instead, the subscribers themselves choose which databases to add to their queries. However, subscribers will be informed about all new databases matching their filter specification. The system has therefore automated much of the resource discovery process.

In this section, we presented the notion of distinct filtering stages within an IS, motivating their definition by retrieval effectiveness considerations. Some of these stages were defined via extended regular expressions. The formal definition of these regular expressions appears in the next section.
