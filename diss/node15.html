These equations assume a collection of messages, which is large enough for the statistically derived parameters to be indicative and useful for feature weighting. However, in contrast to the usual IR searching model, where a small number of query texts is compared to a large, static message collection, the SDI model [Zais76] is normally formulated as that of a sequence of single messages compared individually to a number of queries (figure 2.2). The retrieval techniques typically used are therefore based either on Boolean matching, or, in advanced systems using the vector     space model [Salt91], feature weights are derived from static collections of previously examined messages. In figure 2.2, the size of the messages, which are represented as circles, corresponds to the meaning of a message as modeled by its features. In the SDI model, messages entering the system are filtered through several queries which are represented by different sized holes in the filter. The ``holes'' into which messages can fall are analogous to the features in a query, which represent its meaning. The IR system has a ``net'' inserted in a large message collection to retrieve messages of interest. The mesh of the net represents the query meaning (as described by its features). An IR system uses relationships among the message features in a large collection to facilitate fishing the relevant messages out with a query (net). In section 3.2, we shall propose a more flexible model of dynamic message collections which amalgamates both the SDI and IR paradigms into a hybrid arrangement.
