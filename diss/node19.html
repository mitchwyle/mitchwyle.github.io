Passive Subscriber Model

       

We have seen in section 1.6 that WAN information has become too voluminous to sift manually. Based on the framework and recent developments of Luhn's SDI  model  [Luhn61], [Weav91], [Mond90], [Sedd90] and the informal but visionary software agent   ideas of Alan Kay [Kahl91], [Kay91] we present a new model which assumes a passive subscriber, and a sophisticated, active software system [Wyle89]. In the paradigm of the passive subscriber-model, a centralized service performs routine , algorithmic  information  gathering, filtering   and transmission  tasks. In contrast to the traditional (active) user model of   existing software systems, the passive subscriber model posits a semi-interactive system acting as an intermediary for the user. Semi-interactive in this case means that a user specifies actions for a system to perform periodically or based on asynchronous  events. The model  minimizes direct user-manipulation . It offers the following advantages to the traditional SDI paradigm and other systems:

  • autonomy:   Existing clipping   services and most existing SDI systems require some human  intermediary  action and assistance; our model proposes somewhat more independence on the part of the system, especially for new resource discovery and integration.
  • time dependencies of information need: The passive subscriber model assumes a drift    in user-interest and automatically prods  a user to make profile  changes .
  • integration of new information sources: A subscriber's   profile is automatically matched against new sources as they become available .
  • use of modern IR algorithms: Through the use of dynamic      message collections (described below), the passive user model allows for the application of modern IR methods which employ statistical  text analysis in their homeosemy  calculations. Traditional SDI  models decide whether to send a new information item based solely on the individual item and a stored Boolean  query  in subscriber profiles.

Additionally, this model takes advantage of existing communications channels   that subscribers already understand and use daily. For example, electronic  mail  (email) and client  - server network software     could be used as the communications medium . The subscriber would then receive information items via his or her electronic mail user  agent  (UA)  program and would therefore not have to learn to use a new program or user-interface in order to use a system employing this model. Profile editing can be accommodated by a kind of ``forms processing'' paradigm (see below) or a user-friendlier interactive system. In either case, the emphasis is on using existing, consistent methods of interaction to communicate with the user.

Another new aspect of the passive subscriber model is a method of user - computer interaction whereby each message the system sends to a subscriber    is actually a valid command which can be interpreted by the system. When the system sends information messages, they are surrounded by ``comment'' commands which cause them to be ignored by the system's command interpreter. If L is the set of commands accepted by an information server IS, then the passive subscriber model restricts M, the set of all possible messages sent by the IS to its subscribers to be a subset of L, . When a subscriber asks to see a portion or all of his profile, he or she receives as output the system commands needed to create that portion or the entire profile, respectively. For large profiles, such messages can contain several dozen system commands in a single message. A subscriber can therefore issue commands to the system by making slight changes to the messages the system has sent. This new type of dialog includes fill-in-the-blank forms processing for certain functions typical of interactive systems. But more importantly, an IS can also display comprehensive and detailed sets of examples  in reply to help requests,   expediting the use of complex or difficult system commands via motivating, tutorial palettes of examples. However, since sending pure-text commands to a software system may be incommodious, a user-friendly, interactive  component   for profile editing and update (see section 4.4) can be used to generate the text for the user-dialog component of the passive subscriber model.

A system based on the passive subscriber model (see figure 3.1) accepts commands from the user, creates and stores an interest profile, and then periodically, supplies information to the user. This periodic delivery is automatic; there is no human intermediary. The user passively receives information from the system, without having to re-submit queries every so often. The passive user model accommodates an interest drift, and a system employing the model automatically reminds its subscribers to update their profiles.

In this section, we introduced the passive subscriber model; this model includes an interesting forms-processing paradigm and the concept of palettes of examples. In the next section we describe the individual stages of the filtering process.
