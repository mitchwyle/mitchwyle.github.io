Selective Information Extraction

       

The Pasadena system includes a program for querying remote passive databases. This selective extraction of information ( sei) program runs periodically and scans its database of remote servers. Because the remote database systems have widely varying command languages and query-message requirements, each server has its own query message. The state information stored for each passive source includes:

  • the last query time
  • a (possibly dynamic) message to query the remote database
  • the query interval
  • a log (history) of previous query times
  • a log of response times (and failures)
  • a log of the number of new items returned for specific responses at specific times

The algorithm to query the remote servers is described in [Wyle89] and its implementation is described below (section 4.4). When messages arrive back from the remote databases, the history information for the appropriate remote source database is updated, and new information items are stored in the local Pasadena message cache database. In some cases the query message itself is changed to specify that only new messages be retrieved from the remote database. At certain time intervals, the parameters affecting the query rate for the remote databases (, , ) are re-calculated for each remote system, and the query intervals are changed accordingly.

In addition to passive sources that require Pasadena to query them, the Pasadena system subscribes to several different remote systems that automatically send it messages via email. Pasadena does not have to query these system to receive messages.

Pasadena also gathers information from news feeds periodically by a a general-purpose communications system. This program is an example of a parameter-driven, non-interactive terminal emulator that sends commands to remote interactive systems, and captures the replies from such systems. The ``send and expect script'' is stored as external text, so that changes to the command language or menu structure of the system being queried do not force changes and costly re-compilation of the program. Instead, the expected prompts and commands to be sent can be changed simply by changing the text file. This type of program in conjunction with automatic scheduling systems available from the native operating system is becoming very popular, as such systems form the basis for agents [Kay91]. The program captures new messages which have appeared in the passive remote news system since the last time the program was run. These items appear as one large text, in the form of a ``log'' of the dialog between the querying program and the replies of the system. The captured text is then post-processed by a separate program to convert the message data into a format to be used by the Pasadena system.

This conversion program filters the output of the dialog from newsfeed queries, converting each identified item into a separate message for the Pasadena message database; it then inserts the message into the database in its proper category. The services queried often offer several different newswires; these feeds are further categorized by topic. Each topic of each newswire has a separate Pasadena database category. The Pasadena database system includes methods for the automatic creation, retrieval, and expiration (deletion) of individual messages. The implementation and strategy is similar to the commercial system Clarinet [Temp92] which uses a similar database and distribution medium for commercial newswire services.
